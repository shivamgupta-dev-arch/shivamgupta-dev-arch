## 🏛️ Architecture Focus

```
                    ┌──────────────────────────────────────────────┐
                    │          ENTERPRISE SYSTEM DESIGN            │
                    └──────────────────────────────────────────────┘

  ┌─────────────┐    ┌──────────────┐    ┌─────────────────────────────┐
  │  API Gateway │───▶│  MediatR /   │───▶│    Domain Layer (DDD)       │
  │  (Auth/Rate) │    │  CQRS Bus    │    │    Entities · Aggregates    │
  └─────────────┘    └──────────────┘    └─────────────────────────────┘
                              │                          │
              ┌───────────────┼───────────────┐          ▼
              ▼               ▼               ▼    ┌──────────────┐
       ┌──────────┐   ┌──────────────┐ ┌──────────┐│  Repository  │
       │ Commands │   │   Queries    │ │  Events  │└──────────────┘
       └──────────┘   └──────────────┘ └──────────┘       │
              │               │               │           ▼
              ▼               ▼               ▼    ┌──────────────┐
       ┌──────────────────────────────────────────┐│  SQL Server  │
       │           Application Services           ││  + Dapper   │
       │      Validation · Mapping · Pipeline     │└──────────────┘
       └──────────────────────────────────────────┘
                              │
              ┌───────────────┴──────────────┐
              ▼                              ▼
       ┌──────────────┐             ┌───────────────┐
       │ Apache Kafka │             │     Redis     │
       │  Event Bus   │             │  Cache Layer  │
       └──────────────┘             └───────────────┘
```

> **Philosophy:** Every system I design follows Clean Architecture with CQRS — separating read/write concerns, keeping domain logic pure, and ensuring infrastructure is pluggable. Kafka handles async communication between bounded contexts; Redis eliminates unnecessary DB round-trips.
